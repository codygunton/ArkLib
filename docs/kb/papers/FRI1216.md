# FRI1216

## At A Glance

`FRI1216` is ArkLib's current overview reference for the generalized FRI low-degree test.
It is the main paper page to consult when reading the abstract protocol layer in
`ProofSystem/Fri`.

## What ArkLib Uses From This Paper

- Generalized FRI protocol context for the oracle-reduction formalization in
  [`ArkLib/ProofSystem/Fri/Spec/SingleRound.lean`](../../../ArkLib/ProofSystem/Fri/Spec/SingleRound.lean)
  and the broader `ProofSystem/Fri` subtree.
- Guidance for the repo's support of generalized folding degrees, multi-round composition, and the
  final query round.

## Main ArkLib Touchpoints

- [`ArkLib/ProofSystem/Fri/Spec/SingleRound.lean`](../../../ArkLib/ProofSystem/Fri/Spec/SingleRound.lean)
  cites `FRI1216` directly in the module docstring.
- [`ArkLib/ProofSystem/Fri/Spec/General.lean`](../../../ArkLib/ProofSystem/Fri/Spec/General.lean)
  builds the full FRI oracle reduction as a composition of rounds and is the next file to read
  after the single-round specification.
- The surrounding `ProofSystem/Fri` files package the protocol through domain, round-consistency,
  and security abstractions rather than a single linear exposition.

## Version Notes

- ArkLib currently uses Haböck's summary as the main FRI reference key in the protocol
  specification layer.
- If later PRs depend on a different FRI source lineage or exact theorem numbering, record that in a
  dedicated audit page.

## Known Divergences From ArkLib

- ArkLib expresses FRI as an oracle reduction with explicit round composition, typed oracle
  statements, and protocol parameters.
- The repo aims to cover several generalizations at once rather than freeze the development to the
  narrowest classical FRI presentation.

## Open Formalization Gaps

- The FRI development is broader than this single overview paper.
- If reviewers need a paper-to-code theorem matrix, add a dedicated audit page under
  `docs/kb/audits/` rather than expanding this landing page indefinitely.
